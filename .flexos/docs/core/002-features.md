# Feature Inventory

> A complete list of all features implemented in the new Wimmer Consulting website.

## Core Site Features

- **Responsive Design**: The site layout is fully responsive, providing an optimal viewing experience on desktops, tablets, and mobile devices.
- **Global Navigation**: A sticky header provides consistent, easy access to all main sections of the site from any page.
- **Footer Navigation**: A comprehensive footer includes links to all service pages, company information, and legal documents.
- **SEO Metadata**: Every page includes unique, optimized title tags, meta descriptions, and Open Graph data for social sharing.

## Content & Engagement Features

- **Blog System**: A fully-featured blog built with Astro Content Collections, including a main listing page, individual post pages, and category filters.
- **Case Study Portfolio**: A dedicated section to showcase client success stories, with filtering by industry and service.
- **Newsletter Signup Form**: An integration-ready form allows users to subscribe to an email newsletter, typically placed in the blog and footer.

## Interactive Features

- **Contact Form**: An interactive contact form with client-side validation for required fields (name, email, message) to capture user inquiries.
- **Call-to-Action (CTA) Components**: Reusable CTA sections are placed strategically throughout the site to guide users towards conversion goals like scheduling a consultation.

## Technical & SEO Features

- **Schema Markup**: The site implements structured data (JSON-LD) for LocalBusiness and Service schemas to enhance search engine understanding and visibility.
- **XML Sitemap**: An automatically generated `sitemap.xml` file is available to help search engines efficiently crawl the site.
- **Performance Optimization**: Images are optimized and modern formats are used to ensure fast page load times, a key factor for user experience and SEO.
- **Accessibility (a11y) Standards**: The site is built with semantic HTML and ARIA attributes where necessary to meet WCAG 2.1 AA accessibility requirements.
